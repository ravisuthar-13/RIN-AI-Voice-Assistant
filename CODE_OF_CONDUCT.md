# 📜 Code of Conduct

## Our Commitment

We are committed to creating a respectful, welcoming, and professional
environment for everyone who participates in the RIN project.

Participation may include:

- Reporting bugs
- Suggesting features
- Improving documentation
- Reviewing code
- Submitting pull requests
- Participating in discussions

---

## Expected Behavior

Participants are expected to:

- Communicate respectfully
- Give constructive feedback
- Respect different skill levels
- Be patient with beginners
- Focus discussions on the project
- Respect privacy and security
- Accept technical disagreement professionally
- Help maintain a collaborative environment

---

## Unacceptable Behavior

The following behavior is not acceptable:

- Harassment or discrimination
- Personal attacks
- Insults or threats
- Publishing private information
- Sharing credentials or sensitive data
- Deliberately introducing malicious code
- Spamming discussions or issues
- Misrepresenting the project
- Claiming RIN is production-ready when it is not
- Encouraging unsafe use of experimental features

---

## Security and Privacy Responsibility

Participants must not intentionally submit code that:

- Collects user data without clear consent
- Sends private information to external services
- Contains malware or spyware
- Includes hidden tracking
- Exposes passwords, tokens, or API keys
- Executes destructive commands without clear warnings
- Bypasses security controls
- Accesses files without a valid project purpose

Security concerns should be reported privately according to `SECURITY.md`.

---

## Project Status

RIN is currently under active development.

The project is experimental and not production-ready.

Participants should communicate this status honestly when discussing,
sharing, or contributing to the project.

---

## Enforcement

Project maintainers may remove, edit, or reject:

- Abusive comments
- Unsafe code
- Malicious contributions
- Spam
- Privacy violations
- Misleading project claims

Repeated or serious violations may result in temporary or permanent removal
from project participation.

---

## Reporting Issues

Code of conduct concerns should be reported privately to the project
maintainer.

Reports will be reviewed fairly and handled as confidentially as reasonably
possible.

---

## Scope

This Code of Conduct applies to:

- GitHub issues
- Pull requests
- Discussions
- Documentation
- Project-related communication
- Public representation of the RIN project

---

## Final Note

RIN is being built as a responsible, privacy-aware, and educational AI
assistant project.

Respectful collaboration helps the project improve safely.
